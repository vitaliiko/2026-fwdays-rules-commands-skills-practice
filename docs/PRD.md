# PRD: Воркшоп «Правила, команди та навички для Agentic IDE (Roo Code)»

## 1. Загальна інформація
*   **Тема:** Ефективне налаштування та використання Roo Code як автономного AI-агента.
*   **Тривалість:** 2 години (120 хвилин).
*   **Мова:** Українська.
*   **Інструментарій:** Roo Code (VS Code Extension), Slidev (для презентації), Anthropic/OpenRouter/Local LLMs.

## 2. Мета воркшопу
Навчити учасників переходити від простого чату з ШІ до конфігурування повноцінного автономного агента, який розуміє контекст проекту, дотримується правил та самостійно виконує технічні завдання.

## 3. Програма воркшопу (120 хв)

### Модуль 1: Вступ до Agentic IDE (20 хв)
*   **Що таке Agentic IDE?** Відмінність від Copilot (Auto-complete) та Chat-based асистентів.
*   **Огляд Roo Code:** Архітектура "агент + інструменти". Чому open-source?
*   **Швидкий старт:** Інсталяція, вибір моделі (Claude 3.5 Sonnet як золотий стандарт) та налаштування оточення.

### Модуль 2: Правила (Rules) — Програмування поведінки (40 хв)
*   **Файли `.roorules` та `.clinerules`:** Як задати контекст для конкретного проекту.
*   **System Prompts:** Налаштування глобальної поведінки Roo Code.
*   **Custom Modes:** Створення власних режимів (наприклад, "Security Auditor", "Frontend Architect", "Documentation Bot").
*   **Практика:** Створення файлу правил, який забороняє використовувати певні бібліотеки або навпаки — змушує писати тести перед кодом.

### Модуль 3: Команди та Навички (Commands & Skills) (40 хв)
*   **Автономний термінал:** Як агент використовує `execute_command` для аналізу помилок та запуску тестів.
*   **Tool Use (Skills):** Глибоке розуміння `read_file`, `write_to_file`, `search_files`.
*   **MCP (Model Context Protocol):** Підключення до зовнішніх БД, Slack або Jira (короткий огляд).
*   **Цикли зворотного зв'язку:** Навчання агента "думати" перед виконанням за допомогою правила `explain_before_doing`.

### Модуль 4: Практика — "The Master Agent Challenge" (20 хв)
*   **Сценарій:** Отримання ТЗ -> Агент аналізує код -> Пропонує архітектуру (Architect mode) -> Реалізує фічу (Code mode) -> Виправляє помилки через термінал -> Комітить результат.
*   **Q&A та Висновки:** Чек-ліст для впровадження Roo Code в команду.

## 4. Технічні деталі та налаштування (Settings)
*   **Конфігурація моделі:** Порівняння ціна/якість (API vs Local via Ollama/LM Studio).
*   **Безпека:** Налаштування правил так, щоб агент не передавав секрети (.env) у контекст моделі.
*   **Версіонування правил:** Чому `.roorules` мають бути в Git.

## 5. Очікувані результати
Учасники отримають готовий шаблон `.roorules` та розуміння того, як автоматизувати 60-80% рутинних операцій з кодом через автономні можливості Roo Code.
